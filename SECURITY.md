# Security Policy

## Reporting a Vulnerability

Please report suspected vulnerabilities privately.

Preferred channel:

- GitHub Security Advisories (private report): https://github.com/krishnendusinha/babappai/security/advisories/new

If private reporting is unavailable, open a GitHub issue with minimal detail and request a private follow-up channel.

Do not publish exploit details, proof-of-concept code, or sensitive data in public issues before maintainers have had reasonable time to assess and mitigate.

## Scope

Security reports are in scope when they involve:

- code execution risks in package or CLI behavior
- dependency-chain risks introduced by this repository
- integrity issues affecting release artifacts

Out of scope:

- theoretical issues without reproducible impact
- vulnerabilities only in third-party infrastructure not controlled by this project

## Response Expectations

Maintainers will acknowledge receipt, triage severity, and coordinate remediation and disclosure timing based on impact.
